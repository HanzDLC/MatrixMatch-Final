How to write a good feature description

Describe what each feature lets a user do — not how it would be coded.

  • One sentence, ~20-40 words.
  • Start with the actor and action: "Passenger pays...", "Driver shares...", "User sees...".
  • Plain language — no APIs, endpoints, frameworks, or technical jargon.
  • Don't repeat the label inside the description.
E-wallet integration — Good: "Passenger pays the shuttle fare using e-wallets like GCash or Maya." Bad: "Redirects to GCash API with the booking reference; webhook receives payment status."
Real-time GPS tracking — Good: "Shows the live location of the shuttle to passengers on a map as it moves along its route." Bad: "Driver app POSTs lat/long to backend every 10s; coordinates broadcast over WebSocket."
